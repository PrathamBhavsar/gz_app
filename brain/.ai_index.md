### Token Lifecycle
- Access token (15 min TTL): `accessTokenProvider` (in-memory `StateProvider<String?>`, never persisted)
- Refresh token (7 days): `TokenStorage.saveRefreshToken()` → `flutter_secure_storage`
- On 401: `ApiClient` calls `onRefreshToken` closure → retries once → calls `onLogout` if still 401
- On logout: `POST /auth/logout` → clear all tokens → navigate to `/auth`

### Store Context
- `activeStoreIdProvider` (`StateProvider<String?>`) — in-memory
- Also persisted via `TokenStorage.saveActiveStoreId()`
- Used by all store-scoped API calls (`/stores/:storeId/...`)
- Admin has separate `adminStoreIdProvider` (derived from `adminAuthNotifierProvider`)

### Player WebSocket
- Endpoint: `/ws/users/:userId/notify?token={accessToken}`
- Events: `notification.new`, `session.started`, `session.ended`, `session.extended`, `booking.checked_in`
- **Not yet wired** — needs implementation (see `ux_flow.md` Screen 23)

### Admin WebSocket
- `AdminLiveService` at `lib/core/network/admin_live_service.dart`
- Endpoint: `/ws/stores/:storeId/live?token={accessToken}`
- Auto-connects via `adminLiveServiceProvider` when `adminStoreIdProvider` is set
- Consume via `liveEventsProvider` (StreamProvider<WsEvent>)
